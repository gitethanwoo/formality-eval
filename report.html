Does Prompt Formality Affect AI Effort?

An empirical evaluation of prompt register on model output thoroughness

Abstract We evaluate whether the linguistic register of a user prompt — casual versus formal — affects the thoroughness of AI model outputs on identical tasks. To disentangle tone from instruction content, we employ three conditions: authentic casual (natural brief style), controlled casual (same directives as formal, casual register), and authentic formal (professional with explicit quality expectations). We test 4 models (Claude Haiku 4.5, GPT-5.1 Codex Mini, GPT-5.2 Codex, Claude Opus 4.6) across 3 tasks (copywriting, coding, file organization) with sandboxed tool access and up to 400 agentic steps. We find that the effect is model-dependent and task-dependent: Anthropic models show sensitivity to both tone and instruction content, while OpenAI Codex models are largely insensitive to prompt style on most tasks.

1. Methodology

1.1 Experimental Design

We construct a 4 × 3 × 3 evaluation matrix (36 total runs). Each run provisions a fresh sandboxed environment via bash-tool + just-bash (an in-memory virtual filesystem) pre-loaded with task-specific seed files. Models receive three tools: bash, readFile, and writeFile. The model may take up to 400 steps. We record every step's tool calls, text output, and token usage.

1.2 Three Conditions

The system prompt is held constant across all conditions:

You are a helpful assistant. You have access to tools for reading files, writing files, and executing bash commands in your working directory. Use these tools to complete the user's request.

The independent variable is the user prompt, which varies across three conditions:

  1. Casual (Authentic) — How a casual user would naturally write the request. Less direction, terse, colloquial. Represents the real-world casual use case.
  2. Controlled — Same directives and information content as formal, but written in casual register (lowercase, contractions, informal phrasing). Isolates the effect of tone alone from instruction content.
  3. Formal (Authentic) — How a professional user would write the request. Explicit quality expectations, process instructions, markdown formatting. Represents the real-world formal use case.

The controlled condition enables two comparisons: casual vs. controlled measures the effect of instruction specificity (information gap), and controlled vs. formal measures the effect of register alone (tone gap).

1.3 Laziness Index

We compute a composite laziness index L ∈ [0, 1] where higher values indicate lazier behavior:

L = 0.4(1 − completeness) + 0.3(1 − steps/20) + 0.2(1 − volume/expected) + 0.1(1 − tools_used/4)

Completeness is task-specific: deliverable count for copywriting, test pass rate for coding, sort accuracy for file organization.

3.2 Key Findings

Finding 1: The effect is provider-dependent. Anthropic models (Opus, Haiku) show meaningful laziness deltas between casual and formal prompts on multiple tasks. OpenAI Codex models are largely insensitive to tone on copywriting and coding, though GPT-5.2 Codex shows sensitivity on file-sorting.
Finding 2: Information content matters more than register. The biggest laziness gaps appear between authentic casual (less direction) and formal (more direction). The controlled condition — same information, casual tone — often narrows or closes this gap, suggesting models respond more to what you ask for than how you ask.
Finding 3: Opus responds to register itself on complex tasks. On coding and file-sorting, Opus shows a gradient where controlled casual falls between authentic casual and formal — even though controlled carries the same directives as formal. This suggests that for complex multi-step tasks, the formal presentation (not just content) cues Opus to invest more effort.
Finding 4: Task complexity modulates the effect. Copywriting (a single-shot creative task) shows different sensitivity patterns than coding or file-sorting (multi-step procedural tasks). Models that are insensitive on copywriting may still show tone effects on tasks requiring sustained tool use.

4. Limitations

  • Single trial per condition. Without multiple replications, we cannot distinguish true tone effects from run-to-run stochasticity. No temperature/seed control was applied.
  • Prompt asymmetry. Despite efforts to match information content between controlled and formal conditions, subtle differences in phrasing may still influence behavior (e.g., "don't skip anything" vs. "complete all aspects").
  • Step count as effort proxy. More steps does not necessarily mean better output. A model that takes 40 steps on verification chatter may produce the same quality as one that takes 10 focused steps.
  • Laziness index weights are arbitrary. The 40/30/20/10 weighting was chosen heuristically. Different weightings could change relative rankings.
  • No quality judgment. We measure effort (steps, words, tool calls) but not output quality. A model that writes less may still produce better copy, code, or organization.
